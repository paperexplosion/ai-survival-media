# ブログ記事の管理方法

このディレクトリには、ブログ記事のMarkdownファイルが格納されています。

## 新しい記事の追加方法

1. このディレクトリ（`/src/content/blog/`）に新しい `.md` ファイルを作成します
2. ファイル名は記事のスラッグになります（例: `my-new-article.md` → `/blog/my-new-article`）
3. 以下のテンプレートを使用してください：

```markdown
---
title: "記事のタイトル"
lead: "記事の要約文。一覧ページで表示されます。"
date: "2026-01-27"
category: "🛡️ AI時代の生存戦略・総論"
affiliates:
  - title: "アフィリエイトリンクのタイトル（任意）"
    description: "サービスの説明"
    url: "https://example.com/affiliate-link"
    label: "詳細を見る"
    position: 1
---

## セクション1の見出し

セクション1の本文をここに書きます。

段落は空行で区切ります。

## セクション2の見出し

セクション2の本文をここに書きます。

## セクション3の見出し

最後のセクションです。
```

## Frontmatterフィールド

- `title`: 記事のタイトル（必須）
- `lead`: リード文・要約（必須）
- `date`: 公開日（YYYY-MM-DD形式、必須）
- `category`: カテゴリ名（必須）
  - **有効なカテゴリ**:
    - `🌊 AI時代の本質と世界観`
    - `💣 AIと雇用・労働の崩壊`
    - `🛡️ AI時代の生存戦略・総論`
    - `💼 ハイクラス転職・キャリア戦略`
    - `📖 ストーリーテリング・ドキュメンタリー思考`
    - `🤖 AIと人間の本質・アイデンティティ`
    - `💰 AI時代のお金・財布・経済格差`
    - `🎯 AIを武器にするスキル・思考法`
    - `🏢 組織・会社との付き合い方`
    - `🎨 クリエイティブ・表現者の生存`
    - `⚔️ AI・国家・軍事・倫理`
    - `🧭 キャリア・ポートフォリオ設計`
- `affiliates`: アフィリエイトリンクの配列（任意）
  - `title`: リンクのタイトル
  - `description`: 説明文
  - `url`: リンク先URL
  - `label`: ボタンのラベル
  - `position`: 表示位置（セクション番号）

## 記事の本文

- 見出しは `##` で記述（`#` は使用しない）
- 各見出しがセクションとして扱われます
- Markdownの標準的な記法が使用できます

## ビルドプロセス

1. `npm run dev` または `npm run build` を実行すると、自動的に `scripts/generate-blog-data.mjs` が実行されます
2. このスクリプトがすべての `.md` ファイルを読み取り、`src/lib/blog-posts-data.json` を生成します
3. アプリケーションはこのJSONファイルを読み込んで記事を表示します

## 注意事項

- 記事を追加・編集した後は、開発サーバーを再起動するか、ビルドを実行してください
- ファイル名に使用できるのは英数字、ハイフン、アンダースコアのみです
- 日本語のファイル名は避けてください
