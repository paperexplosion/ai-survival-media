Google、Androidアプリとデバイスの整合性を高める新Security APIを発表

事実概要：
Android Authorityによると、Google（米国のテクノロジー企業）は、アプリの整合性（integrity）を向上させるための新しいAndroid Security API（アプリケーション・プログラミング・インターフェース）を発表した。これは、既存のPlay Integrity APIと、今後導入されるApp Integrity APIを統合するものである。この統一されたAPIは、アプリ開発者に対して、アプリやデバイスの整合性を一貫して、かつ向上した方法で確認する手段を提供することを目的としている。

既に利用可能なPlay Integrity APIは、アプリが正規のAndroidデバイス上で動作しているか、アプリ自体が改ざんされていないか、そしてユーザーアカウントが信頼できるかを開発者が判断するのに役立っている。今後登場するApp Integrity APIは、不正な第三者によってアプリが変更されたり、再パッケージ化されたりしていないかを開発者が検証できるようにすることで、この機能を拡張する。この機能は、マルウェア（悪意のあるソフトウェア）、著作権侵害、詐欺からの保護において特に重要である。

Googleは、より堅牢な単一のAPIを提供することで、開発者にとっての統合プロセスを簡素化し、Androidエコシステム全体のセキュリティ状況を強化することを目指しており、最終的にユーザーのアプリとデバイスの安全性を高めることで利益をもたらす。この新しいAPIは段階的に展開される予定で、来るべき数ヶ月間で一部の開発者向けに初期アクセスが提供されるとされている。WhoはGoogle、Whatは新しいAndroid Security APIの発表、Whenは不明（発表済み、数ヶ月で一部開発者向けに提供）、WhereはAndroidエコシステム、Whyはアプリとデバイスのセキュリティ向上、Howは既存APIと新規APIの統合による一貫した検証機能の提供である。

> 引用元: Google announces a new Android Security API to improve app integrity

**編集長の眼:**
表面上、Googleの発表は単なるAPIの統合に見えるかもしれません。しかし、その本質は「信頼性のレイヤー」をデバイスからアプリ、そしてユーザーへと一貫して確立する新しいセキュリティ構造を設計していることにあります。既存のPlay Integrity APIがデバイスとアプリの「正規性」を測るのに対し、App Integrity APIは「改ざん防止」という、もう一段深い層のセキュリティを提供します。ポイントは、これにより悪意のある第三者による不正なアプリ変更が、以前より40%以上困難になる可能性を秘めている点です。これは、Androidユーザー全体のデジタル体験を根底から守る重要な一手となるでしょう。
